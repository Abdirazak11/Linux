<h2>Package Managers in Linux</h2>

<h3>ğŸ“Œ What is a Package Manager?</h3>
<p>
A package manager is a tool that automates the process of installing, updating, configuring,
and removing software in a Linux system. It ensures that software and its dependencies are
managed efficiently.
</p>

<h3>ğŸ” How Does a Package Manager Work?</h3>

<h4>Repositories (Repos)</h4>
<p>
A package manager fetches software from official repositories (online storage of packages).
<br>
Example: Ubuntu gets packages from <code>archive.ubuntu.com</code>.
</p>

<h4>Installing Software</h4>
<p>When you install software, the package manager:</p>
<ul>
  <li>âœ… Downloads the package from the repository</li>
  <li>âœ… Resolves dependencies (installs additional required software)</li>
  <li>âœ… Installs and configures the software automatically</li>
</ul>

<h4>Updating Software</h4>
<p>
A single command updates all installed packages to the latest version.
</p>

<h4>Removing Software</h4>
<p>
The package manager removes software cleanly without leaving unnecessary files.
</p>

<h3>ğŸŒ How Package Managers Fetch Software from Repositories</h3>
<p>
A repository is a server that stores software packages. When a package manager installs software:
</p>
<ul>
  <li>It checks the repository list (e.g., <code>/etc/apt/sources.list</code> in Ubuntu)</li>
  <li>It downloads the package and its dependencies</li>
  <li>It installs and configures the software automatically</li>
</ul>

<h3>ğŸ”„ Why Should You Run <code>apt update</code> After Installing Ubuntu?</h3>
<p>
Packages included in the Ubuntu ISO image may be outdated. Running the following updates
the package list from repositories:
</p>

<pre><code>sudo apt update
</code></pre>

<p>Then install the latest versions of packages:</p>

<pre><code>sudo apt upgrade -y
</code></pre>

<h3>ğŸ›  Essential Package Manager Commands</h3>

<h4>APT (Debian, Ubuntu)</h4>
<pre><code>sudo apt update         # Update package lists
sudo apt upgrade -y     # Upgrade installed packages
sudo apt install nginx  # Install a package
sudo apt remove nginx   # Remove a package
sudo apt autoremove     # Remove unused dependencies
sudo apt search nginx   # Search for a package
</code></pre>

<h4>DNF (Fedora, RHEL, CentOS)</h4>
<pre><code>sudo dnf check-update   # Check for updates
sudo dnf update         # Update all packages
sudo dnf install nginx  # Install a package
sudo dnf remove nginx   # Remove a package
</code></pre>

<h4>Pacman (Arch Linux)</h4>
<pre><code>sudo pacman -Syu        # Sync and update all packages
sudo pacman -S nginx    # Install a package
sudo pacman -R nginx    # Remove a package
</code></pre>

<h4>Zypper (OpenSUSE)</h4>
<pre><code>sudo zypper refresh        # Refresh package list
sudo zypper update         # Update all packages
sudo zypper install nginx  # Install a package
sudo zypper remove nginx   # Remove a package
</code></pre>
